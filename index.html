Cross‑Sell Propensity Models (PASHA Bank)

Developed ML and rule‑based recommendation models for FX, Foreign Payments, Loans, Salary Cards, and Factoring. For Loans, applied Positive‑Unlabeled Learning to handle scarce negative labels; for FX/FP, used RFM segmentation (Recency, Frequency, Monetary) to classify customers into five tiers: Active, Passive, About to be Passive, Lost, Potentially Active.

Aggregated data from Central Bank, transactions, and customer profiles with time‑windowed features (last 3/6/12 months). Defined CDEs and an extensive feature list, then wrote all features to a DWH datamart. Automated the full flow in Dataiku (ETL → preprocessing → model → DB write‑back) triggered monthly. Outputs feed the internal RM Tool for sales prioritization. Revenue uplift tracked via Tableau and PostgreSQL RM logs, delivering 1.2M+ AZN across 2023–2024.

Fraud Detection & Prevention (PASHA Bank)

Implemented a near real‑time fraud detection framework combining unsupervised anomaly detection (Isolation Forest, Autoencoder) and supervised classification (Gradient Boosting, Logistic Regression) in Dataiku with PySpark for large‑scale data.

Hybrid model performance (test env):

  • Precision: ~88%
  • Recall: ~82%
  • AUC: ~0.91

The supervised model performed best and was moved to production for continuous observation. The system reduced false positives by ~35% and analyst review time by ~25%.

News & NLP Analytics (PASHA Bank)

Built an end‑to‑end news intelligence pipeline for CRM and Counterparty Check. Scrapes trusted Azerbaijani sources every 10 minutes (static via requests + BeautifulSoup, dynamic via Selenium), assigns unique IDs and metadata, and stores to PostgreSQL.

Entity matching: For CRM, allowed 85–90% similarity using FuzzyWuzzy (Levenshtein). For Counterparty Check, applied strict normalization: dictionaries for legal forms like MMC (Mehdud Məsuliyyətli Cəmiyyət — Limited Liability Company) / LLC; transliteration rules (Paşa→Pasha); punctuation/spacing variants (m.m.c / m. m. c); and token co‑occurrence (2–4‑grams) using collections to capture names such as “Bravo”, “Bravo MMC”, or “Bravo Mehdud Məsuliyyətli Cəmiyyəti”. Excluded lowercase common‑word matches (e.g., “bravo” as congratulations).

Exposed an API through Dataiku for Counterparty Check and connected CRM dashboards so RMs see relevant news instantly. Result: RM search time reduced by 30–40 minutes per client; Counterparty Check usage‑driven income up by ~20%.

Operator Email Efficiency Dashboard (PASHA Bank)

Measured First Response Time (FRT), Handling Time (HT), and operator efficiency, plus required staffing. Refactored legacy SQL with modular CTEs, efficient joins, and centralized date logic → ~90% faster queries.

Defined efficiency score: Efficiency = HandlingTime × log(1 + ConversationCount) to reward high throughput while controlling extremes. Metrics honor business time (09:00–18:00), weekends, and holidays. Found Efficiency Factor ≈ 2.9, indicating potential 2–3× staffing optimization without SLA loss.
